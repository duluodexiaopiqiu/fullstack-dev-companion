### 核心建议

#### 何时使用原生开发？
- **1-5个简单页面**：产品介绍、活动页、个人简历、公司官网
- **无复杂交互**：主要是展示内容，少量表单提交
- **性能要求高**：需要最快加载速度
- **快速上线**：1-3天内完成
- **预算有限**：最小技术栈
- **学习目的**：掌握Web开发基础

#### 原生开发的优势
1. **零依赖**：无需安装任何框架或库
2. **性能最优**：无框架运行时开销，文件体积最小
3. **部署最简单**：直接上传HTML/CSS/JS文件到任何服务器
4. **学习成本最低**：直接从Web基础开始学习
5. **完全控制**：可以精确控制每一个细节

#### 避免的误区
1. **不要过度设计**：简单页面不需要复杂框架
2. **考虑实际需求**：根据页面数量和交互复杂度选择技术
3. **性能优先**：静态页面的核心优势是性能
4. **维护成本**：原生代码更容易长期维护

### 使用示例

#### 快速创建静态页面
```bash
# 1. 创建项目
mkdir my-site && cd my-site

# 2. 创建基本文件
touch index.html style.css script.js

# 3. 使用VSCode + Live Server开发
code .

# 4. 参考 static-page-guide.md 获取完整代码
```

#### 在技能中使用
```bash
# 启动技能
npm start

# 选择：开发新手 → 静态页面/简单网站
# 获取推荐路径和特别提示
```
